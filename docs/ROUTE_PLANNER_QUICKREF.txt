【🚀 最快开始 - 复制粘贴运行】

基础用法:
  python3 route_planner.py --start "116.39,39.90" --end "116.46,39.92"

用地名:
  python3 route_planner.py --start "三里屯" --end "国贸"

查看结果:
  open output/route_plan.html

【📍 输入格式说明】

坐标格式 (推荐):
  格式    — lng,lat (经度,纬度，用逗号分隔)
  示例    — 116.39,39.90 (中关村附近)
  识别    — 自动检测，优先级最高

地名格式:
  格式    — 中文地名
  示例    — 三里屯 / 国贸 / 朝阳公园
  调用    — 高德地理编码API
  提示    — 更具体更准确，如"朝阳区三里屯"

────────────────────────────────────────────────────────────────────────────

【🎯 三条路线解读】

路线1 (LEFT):
  特点    — 左侧高热度POI，最优接单
  评分    — 通常最高 (0.20+)
  建议    — 首选方案

路线2 (CENTER):
  特点    — 中间高热度POI，平衡方案
  评分    — 次之 (0.15~0.20)
  建议    — 路线1堵车时选择

路线3 (RIGHT):
  特点    — 右侧高热度POI，备选方案
  评分    — 通常较低 (0.10~0.15)
  建议    — 其他路线都不可行时选

评分含义: 热度总和 / 距离(km)
  0.20+  ⭐⭐⭐⭐⭐ 强烈推荐
  0.15~0.20 ⭐⭐⭐⭐  很好
  0.10~0.15 ⭐⭐⭐    不错
  <0.10   ⭐⭐     较差

────────────────────────────────────────────────────────────────────────────

【⚙️  完整参数】

必需:
  --start "起点"     起点(坐标或地名) [必需]
  --end "终点"       终点(坐标或地名) [必需]

可选:
  --city beijing     城市(默认beijing)
  --output file      输出地图文件(默认output/route_plan.html)
  --poi-data file    POI数据文件(默认output/poi_heat_weights.json)

示例:
  python3 route_planner.py \
    --start "116.39,39.90" \
    --end "朝阳公园" \
    --output my_routes/route1.html

【📊 输出内容说明】

控制台输出:
  ✓ 三条路线详情
    - 途经点数
    - 热度总和
    - 距离 (km)
    - 时长 (分钟)
    - 接单概率评分

  ✓ 每条路线对应的途经POI名称

HTML地图:
  ✓ 交互式Folium地图
  ✓ 三条不同颜色的路线
    - 红色 = 路线1 (LEFT)
    - 青色 = 路线2 (CENTER)
    - 黄色 = 路线3 (RIGHT)
  ✓ 起点(绿色) + 终点(红色)
  ✓ 途经POI(彩色圆点)

【❓ 常见问题】

Q: 只规划出1-2条路线?
A: 演示数据较少。运行poi_heat_analyzer.py生成真实数据

Q: 地名无法识别?
A: 尝试更具体的名字，或改用坐标

Q: API错误?
A: 检查.env中的AMAP_API_KEY是否正确

Q: 地图打不开?
A: 确保output目录存在，或用浏览器直接打开HTML文件

Q: 如何改变路线数量?
A: 编辑route_planner.py中的 sides = ["left", "center", "right"]
