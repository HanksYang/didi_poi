【📸 上传什么？】

支持的文件：
  - 格式   — JPG / PNG / GIF
  - 大小   — 最大 50MB
  - 内容   — 任何热力图（需要红色表示热度）

热力图来源：
  ✓ 高德地图截图（推荐）
  ✓ 系统自定义热力图
  ✓ 其他地图软件截图（需要红色表示热度）

────────────────────────────────────────────────────────────────────────────

【⚙️ CLI 完整参数】

python3 heatmap_uploader.py \
  --image <文件路径>           [必需] 热力图文件
  --period <时段>              [可选] morning_peak/daytime/evening_peak/auto(推荐)
  --sigma <数值>               [可选] 高斯衰减半径(km)，默认3
  --db <路径>                  [可选] SQLite数据库，默认data/traffic.db
  --poi-json <路径>            [可选] POI JSON文件，默认output/poi_heat_weights.json

示例：
  # 基础
  python3 heatmap_uploader.py --image heatmap.jpg

  # 完整
  python3 heatmap_uploader.py \
    --image heatmap.jpg \
    --period daytime \
    --sigma 3.5

────────────────────────────────────────────────────────────────────────────

【🎛️ Web 界面参数】

时段选择:
  ├─ auto           → 根据当前时间自动判断（推荐）
  ├─ morning_peak   → 早高峰 (07:00-09:00)
  ├─ daytime        → 白天 (10:00-16:00)
  └─ evening_peak   → 晚高峰 (17:00-20:00)

Sigma 滑块 (0.5-10km):
  ├─ 0.5-1km        → 快速衰减，中心权重集中
  ├─ 3km（推荐）    → 平衡，通用场景
  └─ 5-10km         → 缓慢衰减，广域覆盖

────────────────────────────────────────────────────────────────────────────

【🧮 高斯衰减公式】

weight = exp(-d² / 2σ²)   [d=距离(km), σ=sigma参数]

权重表（σ=3km）:
  距离    权重    说明
  0km    1.00    热度中心
  1km    0.88    接近中心
  3km    0.61    σ处（~60%）
  5km    0.29    明显衰减
  8km    0.07    几乎无贡献

【📊 数据写入】

更新位置1: SQLite traffic_samples 表
  表字段: id, sampled_at, period, grid_lng, grid_lat, score
  操作: INSERT 新行，不覆盖历史数据
  特点: 可与既有路况数据共存，用于聚合分析

更新位置2: output/poi_heat_weights.json
  字段: rank, name, heat_score（更新）
  操作: 覆盖式更新，重排rank
  特点: 供路径规划系统实时调用

流程图:
  识别地点 (name, lng, lat, heat_score)
    ↓
  Step 1: 名称匹配
    (如"三里屯" ⊂ "三里屯SOHO" → 匹配)
    ↓ 未匹配
  Step 2: 距离匹配
    (如距离 < 500m → 匹配)
    ↓ 未匹配
  Step 3: 新增POI
    (作为新的POI加入)

【❌ 常见错误】

"ANTHROPIC_API_KEY not found"
  → 检查 .env 是否包含有效的 API Key
  → echo $ANTHROPIC_API_KEY  # 或者
  → cat .env | grep ANTHROPIC

"没有检测到明显的热度区域"
  → 图片中红色像素不足
  → 上传一张真正的热力图（红色代表热度）
  → 尝试亮度调整或颜色增强

"Claude返回的JSON解析失败"
  → API Key 配额用尽（检查console.anthropic.com）
  → API Key 无效或权限不足
  → 网络连接问题

"地点无法匹配"
  → 识别的地点名称可能与POI名称差异大
  → 可编辑 poi_heat_weights.json 手动修正
  → 调整匹配距离阈值（目前 500m）

"Web页面打不开"
  → 端口 5000 被占用，改用其他端口：
    python3 web_app.py --port 8080
  → 检查防火墙是否阻止

【📱 使用场景】

场景1: 定时热力图更新
  for h in {0..23}; do
    wget https://map.api/heatmap_$h.jpg
    python3 heatmap_uploader.py --image heatmap_$h.jpg
  done

场景2: 手动热力图调整
  1. 从高德地图截屏
  2. python3 web_app.py
  3. 拖拽上传，查看预览
  4. 调整 sigma 参数
  5. 点击上传

场景3: 与路径规划系统集成
  1. 上传热力图
  2. poi_heat_weights.json 自动更新
  3. route_planner.py 自动读取新权重
  4. 用户查询时得到最新路线

場景4: 数据分析
  # 查询日均热度变化
  sqlite3 data/traffic.db "
    SELECT date(sampled_at) as day, period, AVG(score)
    FROM traffic_samples
    GROUP BY day, period
    ORDER BY day DESC;
  "
